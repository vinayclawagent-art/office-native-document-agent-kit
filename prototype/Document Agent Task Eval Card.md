## Workflow sequence
1. Pick one real document job with a visible output.
2. Run it in the office-native plugin and in a baseline chat workflow.
3. Score context retention, citation/source quality, diagram usefulness, and edit friction.
4. Capture screenshots or exported docs as evidence.
5. Decide whether this is a product wedge, feature benchmark, or not-now signal.

## Decision gate
Choose exactly one after evidence is attached:
- [ ] Promote into a stronger reusable workflow.
- [ ] Pilot-only: useful, but needs one more trial.
- [ ] Iterate: keep package active and patch the prototype.
- [ ] Hold: evidence too thin or workflow not worth repeating.
